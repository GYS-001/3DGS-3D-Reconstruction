#  3D Gaussian Splatting 真实物体高精度3D重建工程
> 独立完成端到端3D重建全链路 | 攻克CUDA算子编译 | 工程化落地自定义场景

##  项目概述
本项目基于**3D高斯溅射（3DGS）** 实现真实物理世界物体的高精度3D重建，独立打通 **视频采集 → 多视图几何稀疏重建 → AI驱动高斯参数优化 → 可微分渲染 → Web3D交互部署** 全流程工程闭环。

突破传统几何重建纹理缺失、鲁棒性差的痛点，通过**监督式机器学习优化百万级3D高斯参数**，生成可实时渲染的高精度3D数字资产；全程解决Windows平台CUDA/C++算子编译、多工具链兼容、显存约束等工业级工程难点，实现零模板依赖的自定义场景重建。

##  核心技术栈
| 技术方向 | 具体栈 |
| ---- | ---- |
| 算法框架 | 3D Gaussian Splatting、多视图立体匹配(MVS)、可微分渲染 |
| 工程工具 | PyTorch、CUDA 11.8、C++自定义算子、COLMAP、FFmpeg |
| 部署展示 | PlayCanvas Web3D、H.264高清视频编码 |
| 开发环境 | Windows 10、Anaconda、MSVC v142编译工具链 |

##  全流程工程实现（端到端闭环）
### 1. 自定义数据集工程化采集与预处理
采用环绕式稳定拍摄方案，基于FFmpeg恒定帧率抽帧，生成**220张无模糊、多角度对齐**的训练图像，构建符合3DGS规范的标准化数据集。

### 2. COLMAP 稀疏几何重建（GPU加速）
基于SIFT特征提取+暴力匹配完成多视图特征关联，启用GPU加速优化计算效率；约束**PINHOLE针孔相机模型**，自动标定相机内参，生成**13901个高精度稀疏3D点云**与相机位姿矩阵。

### 3. 多工具链数据格式兼容适配
完成COLMAP二进制模型向3DGS文本格式的无损转换，校验 `cameras/images/points3D` 三大核心文件几何参数一致性，彻底解决跨工具数据解析失败问题。

### 4. 3D高斯模型AI训练与收敛调优
基于Adam优化器实现百万级高斯参数（位置、旋转、颜色、透明度）监督式学习，以**L1+SSIM**为损失函数，迭代30000轮完成模型收敛；针对小物体场景参数适配，保障纹理与几何结构高精度还原。

### 5. 可微分渲染与多端成果部署
批量渲染360°全视角图像序列，适配FFmpeg H.264编码规范生成高清展示视频；将PLY高斯模型部署至Web端，实现**无插件实时交互（旋转/缩放/漫游）**。

##  核心技术难点 & 工程化解决方案（硬核核心）
### 1. Windows平台 3DGS CUDA/C++ 自定义算子编译
**难点**：3DGS核心光栅化算子依赖CUDA+C++混合编程，Windows下存在PyTorch-CUDA版本绑定、MSVC工具链兼容、Git子模块缺失、动态库加载失败等连锁编译问题。
**解决方案**：
- 严格对齐版本链：CUDA 11.8 + PyTorch 2.0.1 + MSVC v142，消除ABI不兼容
- 补全 `diff-gaussian-rasterization与simple-knn` 子模块依赖，打通编译链路
- 适配消费级显卡约束，完成算子无报错编译与Python绑定

### 2. 稀疏重建数据校验与格式兼容
**难点**：COLMAP默认二进制模型与3DGS文本格式不兼容，易出现相机内参丢失、点云索引错位。
**解决方案**：原生文本导出+拓扑一致性校验，实现零数据丢失，训练数据100%适配。

### 3. 消费级显卡显存约束优化
**难点**：原生高分辨率训练显存占用＞10GB，6-8GB显卡直接OOM崩溃。
**解决方案**：分辨率自适应缩放，显存占用降低60%；聚焦物体核心区域优化，杜绝无效算力浪费。

### 4. 渲染管线编码兼容适配
**难点**：奇数分辨率导致FFmpeg H.264编码失败。
**解决方案**：分辨率自动归一化处理，无损适配编码规范，批量生成高清视频。
